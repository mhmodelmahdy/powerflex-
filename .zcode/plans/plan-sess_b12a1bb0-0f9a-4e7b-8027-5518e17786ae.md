## الخطة: تحويل المشروع لـ standalone جاهز للرفع على Vercel

التطبيق الحقيقي هو `artifacts/pf753-guide` بس (React + Vite + Tailwind، دليل PowerFlex 753 بالعربي RTL، داتا ثابتة في `src/data/parameters.ts`). كل الباقي سكافولد Replit مش مستخدم. هحوّله لمشروع npm مستقل في جذر الـ repo.

### المرحلة 1: نقل محتوى pf753-guide لجذر المشروع
- أنقل كل ملفات `artifacts/pf753-guide/` (src, public, index.html, components.json, vite.config.ts) لجذر الـ repo.
- أتجنّب التعارض: الجذر فيه `package.json`, `tsconfig.json`, `.gitignore` قديمة — هتعتمد على ملفات pf753-guide.

### المرحولة 2: حذف سكافولد Replit غير الضروري
أشيل المجلدات/الملفات دي:
- `artifacts/` (بعد نقل pf753-guide)
- `lib/` (api-client-react, api-spec, api-zod, db — غير مستخدمة)
- `scripts/` (post-merge, tsconfig.tsbuildinfo)
- `attached_assets/` (صور/PDF مرجعية مش مستخدمة في الكود)
- `.agents/`, `.local/` (ملفات Replit/IDE)
- ملفات Replit: `.replit`, `.replitignore`, `replit.md`, `.npmrc`, `pnpm-workspace.yaml`, `pnpm-lock.yaml`, `tsconfig.base.json`, `tsconfig.json` (القديم)

### المرحلة 3: ظبط package.json مستقل (npm)
- أشيل `preinstall` hook (اللي بيفرض pnpm).
- أبدّل كل `"catalog:"` و `"workspace:*"` بإصدارات فعلية من `pnpm-workspace.yaml` (مثلاً react `19.1.0`, vite `^7.3.2`, tailwindcss `^4.1.14`, إلخ).
- أشيل `"@workspace/api-client-react"` (مش مستخدم) و `"@replit/vite-plugin-*"` (هتشيلها من vite config).
- أضيف scripts: `dev`, `build`, `preview` (بدل serve), `typecheck`.
- أضيف `"engines": { "node": ">=20" }`.
- ألمّ قائمة الـ deps الحقيقية (radix-ui المستخدمة + الأدوات الفعلية).

### المرحلة 4: تبسيط vite.config.ts
- أشيل إلزامية `PORT` و `BASE_PATH` (دي بتعمل throw وتمنع البناء).
- أشيل `runtimeErrorOverlay` و بلوك cartographer/devBanner (Replit-only).
- أبقي `@` alias لـ `src`، وأشيل `@assets` alias.
- أحدد `base: './'` (آمن)، و output لـ `dist`.
- أبقي `server.host: '0.0.0.0'` و port 5000 افتراضي.

### المرحلة 5: ظبط tsconfig.json مستقل
- ما يبقاش بيشير لـ `../../tsconfig.base.json`. أكتب tsconfig مكتفي ذاتياً (compilerOptions: jsx, lib, moduleResolution, paths لـ `@/*`) من غير references خارجية. أشيل الـ reference لـ `api-client-react`.

### المرحلة 6: إعداد Vercel
- أضيف `vercel.json` فيه:
  - `buildCommand: "npm run build"`
  - `outputDirectory: "dist"`
  - `rewrites: [{ "source": "/(.*)", "destination": "/index.html" }]` (عشان SPA routing بتاع wouter يشتغل على كل المسارات).
- Vercel هيكتشف Vite تلقائياً، لكن vercel.json بيضمن الـ rewrites.

### المرحلة 7: تحديث .gitignore
- أبقي قاعدة `node_modules`, `dist`, `.env`, إلخ.
- أشيل قواعد Replit المحددة لو مش محتاجين، أضيف `.env*`.

### المرحلة 8: التوثيق
- أكتب `README.md` بسيط بالعربي: تعريف المشروع، أوامر `npm install`/`npm run dev`/`npm run build`، وطريقة الرفع على Vercel.

### المرحلة 9: التحقق
- أشغّل `npm install` للتأكد إن الـ deps بتتسطّب من غير أخطاء.
- أشغّل `npm run build` للتأكد إن البناء بيتم بنجاح.
- لو فيه أخطاء imports فائتة (مثلاً مكون شيلته بالغلط) هصلّحها.

**النتيجة:** مشروع نظيف، مستقل، يشتغل بـ `npm run dev` محلياً ويرفع على Vercel مباشرة من GitHub من غير أي إعدادات إضافية.
